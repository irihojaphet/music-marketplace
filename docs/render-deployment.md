# Render Deployment

This project is prepared for deployment on Render with:

- a Python web service for FastAPI
- a static site for the React frontend
- a managed Render Postgres database
- an infrastructure definition in `render.yaml`

Official references used:

- FastAPI on Render: https://render.com/docs/deploy-fastapi
- Static sites: https://render.com/docs/static-sites
- Render Postgres connection URLs: https://render.com/docs/databases
- Blueprint YAML reference: https://render.com/docs/blueprint-spec

## Recommended Deployment Shape

### Backend

- Service type: `Web Service`
- Runtime: `Python`
- Root directory: `backend`
- Build command: `pip install -r requirements.txt`
- Pre-deploy command: `python -m alembic upgrade head`
- Start command: `python -m app.db.seed && uvicorn app.main:app --host 0.0.0.0 --port $PORT`

### Frontend

- Service type: `Static Site`
- Root directory: `frontend`
- Build command: `npm ci && npm run build`
- Publish directory: `dist`
- SPA rewrite: `/* -> /index.html`

### Database

- Service type: `Render Postgres`
- Connect the backend with the database's internal connection string

## Deploy With Blueprint

1. Push this repository to GitHub.
2. In Render, open `Blueprints`.
3. Create a new blueprint instance from this repository.
4. During the first sync, provide values for the `sync: false` variables:
   - `SECRET_KEY`
   - `FRONTEND_URLS`
   - `VITE_API_BASE_URL`

## Environment Variable Values

### Backend

- `SECRET_KEY`
  - Use a long random value generated by a password manager or secret generator.
- `DATABASE_URL`
  - Provided automatically from the Render Postgres instance in `render.yaml`.
- `FRONTEND_URLS`
  - Use a comma-separated list.
  - Example:
    `https://music-marketplace-web.onrender.com,https://your-custom-domain.com`

### Frontend

- `VITE_API_BASE_URL`
  - Set this to the public Render URL of the backend.
  - Example:
    `https://music-marketplace-api.onrender.com`

## Recommended First Production Checklist

1. Deploy the blueprint.
2. Open the backend service and confirm `/health` responds successfully.
3. Open the frontend and verify:
   - marketplace loads
   - search works
   - registration works
   - login works
   - purchase flow works
   - library and rating flow work
4. Open the backend docs page at `/docs` and confirm the live API is reachable.
5. If you add custom domains, update `FRONTEND_URLS` to include them.

## Practical Notes

- Render web services must bind `0.0.0.0` and use `$PORT`.
- Render Postgres exposes both internal and external URLs; the backend should use the internal URL when running in the same Render region.
- Static sites are served over Render's CDN and should use a rewrite rule for client-side React routing.
- `app.db.seed` is idempotent in this project. It exits without changing data if users already exist.

## Suggested Production Polish
